# OpenAI Provider Feature Parity

Sources used: OpenAI Responses, Chat Completions, conversation-state, reasoning, function-calling, and hosted-tool docs; local AI SDK OpenAI provider sources under `.context/attachments/extracted/ai-main/packages/openai/src`.

| Feature | Provider docs behavior | AI SDK behavior | SwiftAgent current behavior | Gap | Priority | Tests needed |
| --- | --- | --- | --- | --- | --- | --- |
| Chat Completions text | `chat/completions` accepts messages and returns assistant content. | Supports chat text generation. | Supported by `OpenAILanguageModel(apiVariant: .chatCompletions)`. | None for basic text. | Done | Existing request/response replay. |
| Chat Completions structured output | Uses `response_format` / JSON schema where supported. | Supports structured output. | Sends JSON schema response format and decodes assistant content. | Provider/model-specific unsupported cases are not exhaustively covered. | Medium | Request body + refusal/error fixture. |
| Chat Completions local tools | Sends function tools and receives `tool_calls`. | Supports tool calls and tool results. | Supports non-streaming local tool calls and tool-result replay. | Chat streaming tool-call assembly is not implemented. | High | Streaming chat tool-call fixture. |
| Responses text | `responses` accepts item-based input and returns output items / `output_text`. | Supports Responses text. | Supported by `.responses`. | None for basic text. | Done | Existing response fixture. |
| Responses streaming text | SSE emits output text deltas and completion events. | Emits typed stream parts. | Supports text start/delta/completion events. | Event coverage should be checked against latest docs. | Medium | Full SSE replay with completion metadata. |
| Responses structured output | Uses `text.format` JSON schema. | Supports structured output. | Sends schema and decodes text/output JSON. | No matrix-confirmed coverage for strict/refusal edge cases. | Medium | Structured success + refusal fixture. |
| Responses local function tools | Uses `function_call` output items and `function_call_output` inputs. | Supports item replay, tool call IDs, and function outputs. | Supports non-streaming and streaming function-call argument assembly. | Needs stronger request tests for exact item replay after tool output. | High | Multi-turn function call/output replay. |
| Manual context replay | Prior output items, including reasoning/tool items, can be passed in next input. | Primary input-construction path. | Preserves raw output/tool/reasoning metadata and replays known items. | Coverage is incomplete for all Responses item kinds. | High | Request-body tests for message, function, reasoning, compaction. |
| `store` | Responses includes `store`; OpenAI defaults stored behavior unless `store: false`. | Defaults `store` to true for Responses unless overridden. | `store` exists in custom options and is sent when set; omitted otherwise. | SwiftAgent does not currently decide a canonical default in code. | Medium | Request tests for omitted/true/false. |
| `previous_response_id` | Continues from a previous stored response; cannot be used with `conversation`. | First-class provider option. | Not first-class. | Missing option and request mapping. | High | Request test and validation conflict with conversation. |
| `conversation` | Uses OpenAI Conversations state; cannot be used with `previous_response_id`. | First-class provider option. | Not first-class. | Missing option and request mapping. | High | Request test and validation conflict with previous response ID. |
| `include` | Adds fields such as `reasoning.encrypted_content`, file search results, logprobs. | First-class option; AI SDK auto-adds some includes. | Not first-class; may be possible through `extraBody`. | Missing typed option and automatic includes. | High | Request tests for explicit and auto include. |
| Encrypted reasoning | Stateless/ZDR reasoning continuity requires `include: ["reasoning.encrypted_content"]`. | Auto-adds encrypted reasoning include for `store === false` reasoning models; preserves encrypted content. | Parses/preserves encrypted content when provider returns it. | Does not auto-request encrypted content. | High | Reasoning request + continuation fixture. |
| Reasoning summaries/items | Reasoning items can include summaries/content/encrypted content. | Maps reasoning parts and metadata. | Parses reasoning output items into transcript reasoning entries. | Needs coverage for all reasoning item shapes and stream variants. | High | Non-streaming and streaming reasoning fixtures. |
| Compaction | `/responses/compact` returns encrypted compaction items for future input. | Models compaction metadata. | No first-class compaction API or full item handling. | Missing. | Medium | Request/response fixture after API added. |
| Hosted web search | Hosted tools can produce web search calls, sources, and citations. | Models OpenAI hosted web-search tool parts and auto-includes sources. | No first-class hosted web-search support. | Missing. | High | Hosted tool stream fixture after API added. |
| Hosted file search | Hosted file search can return results/citations with include flags. | Models file-search tool parts and metadata. | No first-class hosted file-search support. | Missing. | Medium | Hosted file-search fixture after API added. |
| Code interpreter/container tools | Code interpreter emits tool calls, code deltas, outputs, container IDs. | Models code-interpreter parts and auto-includes outputs. | No first-class code-interpreter support. | Missing. | Medium | Stream fixture after API added. |
| Citations/annotations/logprobs | Output text may include annotations; logprobs require include/options. | Maps annotations, citations, source metadata, logprobs. | Not fully modeled. | Missing provider metadata normalization. | Medium | Decode fixture for annotations/logprobs. |
| Usage/HTTP metadata | Responses and Chat return usage, IDs, headers/rate-limit metadata. | Preserves response metadata. | Preserves usage, request IDs, provider/model metadata, headers. | Need latest header/rate-limit audit. | Low | Header metadata fixture. |
